# 暑期日历 H5

这是一个给爷爷奶奶在微信里查看的轻量日历应用。直接打开 `index.html` 就能预览；正式给家人使用时，把整个 `summer-calendar` 文件夹放到任意 HTTPS 静态托管服务上，再把链接发到微信群即可。

当前 GitHub Pages 发布版本默认包含截图录入的 2026 年暑期行程，版本为 `2026-summer-xueersi-travel-v2`，共 43 条安排。后续更新 `schedule-data.js` 并重新推送后，家人打开同一个链接就能看到最新版。

## 主要功能

- 今天安排、下一项提醒、月历和全部清单
- 按课程、出行、家庭、提醒筛选
- 每项日程支持建议出发时间、地点、联系人、电话、物品和备注
- 支持勾选完成、编辑、新增、删除
- 支持腾讯地图 URI 导航、拨打电话、复制地点
- 支持导出 JSON、导入 JSON、复制带数据的分享链接和导出 `.ics` 日历文件

## 替换真实日程

可以在页面的「管理」里新增和编辑，也可以直接改 `schedule-data.js`。页面编辑会保存在当前浏览器本地；如果要换手机继续维护，先在「管理」里导出 JSON，再在另一台设备导入。

数据格式如下：

```js
{
  id: "2026-07-01-art",
  type: "course",
  title: "创意美术课",
  date: "2026-07-01",
  start: "14:00",
  end: "15:30",
  leaveTime: "13:30",
  location: "少年宫 3 楼美术教室",
  address: "真实地址",
  contactName: "王老师",
  contactPhone: "13800000000",
  bag: ["画具袋", "围裙", "水杯"],
  notes: "作品当天带回。"
}
```

`type` 可选：`course`、`trip`、`family`、`todo`。

## 发到微信

微信不能直接打开你电脑上的 `file://` 本地路径。要给家人用，需要把整个文件夹部署成公网 HTTPS 链接。

可选方式：

- GitHub Pages：适合免费公开分享，把文件夹内容上传到一个仓库并开启 Pages。
- Vercel / Netlify：适合拖拽上传静态文件夹，自动生成 HTTPS 链接。
- 腾讯云 CloudBase / 对象存储静态网站：国内访问更稳，适合后续绑定自己的域名。

分享方式有两种：

- 固定链接：更新 `schedule-data.js` 后重新部署，家人打开同一个网址就是最新版。
- 快照链接：部署后在页面「管理」里点「复制微信分享链接」，链接会带上当前数据，适合临时发一版日程。

如果以后需要所有家庭成员实时同步同一份数据，就需要再加一个后端数据库；当前版本是静态 H5，优点是轻、简单、容易发链接。
